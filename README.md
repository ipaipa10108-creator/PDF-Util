# Local PDF Web Suite (純本地端網頁 PDF 綜合處理工具)

![Privacy](https://img.shields.io/badge/Privacy-100%25%20Local-emerald?style=flat-square)

一個現代化、高質感的網頁端 PDF 綜合編輯與電子簽署工具。

### 🔒 絕對的隱私與安全承諾
本工具為 **100% 靜態前端網頁應用**。所有的 PDF 解析、渲染、頁面編輯、裁切與簽章生成皆在您的瀏覽器本地沙盒中完成。**100% 承諾不將任何文件、個資或簽名圖像上傳至任何後端伺服器**，保障極致的文件隱私安全。

* **操作方式**：點擊頂部工具列（或手機底部懸浮列）的「**插入頁面**」按鈕，即可開啟插入彈窗。
* **自訂範圍**：支援載入另一個 PDF。可選擇插入全部頁面，或使用語法輸入框輸入自定義頁碼（如 `1, 3-5`，代表載入該 PDF 的第 1 頁與第 3 到第 5 頁，若語法錯誤將會出現紅字提示）。

### 1. 📂 插入外部 PDF 頁面
* **操作方式**：點擊頂部工具列（或手機底部懸浮列）的「**插入頁面**」按鈕，即可開啟插入彈窗。
* **自訂範圍**：支援載入另一個 PDF。可選擇插入全部頁面，或使用語法輸入框輸入自定義頁碼（如 `1, 3-5`，代表載入該 PDF 的第 1 頁與第 3 到第 5 頁，若語法錯誤將會出現紅字提示）。
* **定位插入**：可從下拉選單中指定插入至主文件的「最前面（第 1 頁之前）」或「第 N 頁之後」。
* **防偏移對齊**：所有已放置的電子簽章皆綁定唯一的 `pageId`，即使隨後任意插入或刪除頁面使頁面順序重排，簽章依然會精準留在正確的頁面上，不會發生任何錯位。

### 2. 🎨 電子簽章手寫優化與粗細調整
* **操作方式**：點擊頂部工具列（或手機底部懸浮列）的「**簽章**」按鈕以開啟電子簽章管理員。
* **畫筆屬性調整**：在手寫簽名分頁中，支援選擇「黑、藍、紅」三種筆跡顏色，並提供「細 (2px)」、「中 (4px)」、「粗 (6px)」三種筆劃粗細。
* **防擦除邏輯**：最佳化了 Canvas 初始化機制，在手寫過程中切換顏色或粗細**絕對不會**清空或擦除您已經寫好的一半簽名。

### 3. 📱 手持裝置觸控事件優化
* **操作方式**：在手機或平板的瀏覽器上，載入 PDF 並開啟簽署模式。
* **觸控拖曳與縮放**：直接使用手指觸碰縮圖上的簽名即可任意拖曳移動；拉動右下角的縮放手柄即可進行等比例大小縮放。
* **網格防滾動干擾**：當手指正在拖曳或縮放簽章時，系統會自動攔截並暫停背景網頁的滾動，保證手指操作精準流暢。

### 4. 🎛️ 手機端專屬懸浮操作列 (響應式 UI)
* **自動分流**：在寬度小於 768px（行動端）時，頂部 Header 自動將複雜的編輯控制項隱藏，只保留 Logo、下載、分享與深色模式。
* **拇指熱區**：載入 PDF 後，頁面底部會浮現一條磨砂玻璃質感的懸浮控制列，提供「全選/清空」、「旋轉 90°」、「刪除」、「插入頁面」與「簽章」等快捷按鈕，方便行動端用戶以單手輕鬆點擊操作。

### 5. 📥 PWA 檔案分享接收器 (Web Share Target API)
* **註冊分享目標**：本 PWA 已註冊為系統檔案分享接收器。
* **操作方式**：在手機上將此 PWA 應用程式「**新增至主畫面**」安裝後。在 LINE、檔案管理器、Gmail 等其他 App 中選擇分享 PDF 檔案至應用程式時，直接在分享選單中點選 **Local PDF**，系統即可自動開啟本程式並載入該 PDF，省去手動開啟與尋找檔案的步驟。
* **技術實現**：在背景透過 Service Worker 攔截系統發起的 POST 檔案分享請求，暫存至瀏覽器快取儲存空間中，隨後重導向至網頁首頁自動加載，是純前端靜態部署下的完美 PWA 分享解決方案。

### 6. 🔄 頁面順序重排與指定位置移動
* **滑鼠拖曳排序**：在主視窗中，可直接拖動頁面的縮圖，任意微調或重新編排 PDF 頁面的先後順序。拖放時網格會以虛線高亮提示放置點，提供即時動態反饋（在電子簽章放置模式下會自動禁用拖曳以防干擾）。
* **指定位置移動**：每頁縮圖下方新增「**移動**」按鈕，點擊可喚起一個簡緻的行內輸入框。您可以輸入目標頁碼，將該頁精準移至「第 X 頁之後」或「最前面（輸入 0，代表移至第 1 頁之前）」，為長篇幅 PDF 文件帶來極佳的編輯便利性。

### 7. 快速圖章與動態產生器
- 提供 6 款常見的實用合約/簽核圖章：
  1. **樣本 (Sample)**
  2. **要保人 (簽名)**
  3. **被保險人 (簽名)**
  4. **小孩 (簽名)**
  5. **勾勾 (提供四種不同的打勾樣式)**
  6. **請勿列印使用本範例文件!**
- 支援底色快速切換：**透明 (預設)、紅色、藍色、黑色、黃色**。
- 圖章採用動態 Canvas 實時渲染與導出高解析度透明 PNG，邊框與字體皆經精細化排版。
### 8. 底圖透明貼紙張貼與複製
- **全部張貼**：一鍵將當前頁面的貼圖，複製到其餘所有頁面的相同相對位置。
- **指定頁面張貼**：開啟頁面選擇器（含縮圖預覽），供使用者複選特定頁面進行批次張貼。
### 9. 進階雙擊操作與齒輪設定
為解決在貼紙縮得很小時，控制按鈕互相重疊且易誤觸的問題，我們推出了全新的**操作模式設定**（置於頂部工具列齒輪按鈕內）：
- **懸浮顯示控制按鈕 (預設)**：Hover 貼紙時，於四角顯示「複製」、「指定」、「刪除」以及「等比例縮放」按鈕。
- **雙擊貼圖彈窗詢問**：貼圖表面保持乾淨（不顯示操作按鈕，保留右下角小縮放手柄），透過雙擊（Double Click / Double Tap）貼紙彈出大按鈕選單（包含全部頁面張貼、指定頁面張貼、刪除貼圖），對小尺寸貼圖與手機操作非常友善。
### 10. 撤銷與重做 (Undo / Redo) 功能
- 完整記錄所有頁面操作（旋轉、刪除、排序、插入頁面、裁切等）與貼圖狀態。
- 支援最大 30 次歷史快照。
- **防抖快照機制**：貼紙拖曳或縮放時，於「按下瞬間 (onMouseDown / onTouchStart)」以及「點擊刪除前」立即記錄，避免移動時產生大量冗餘快照。
- 頂部欄設有撤銷與重做按鈕，手機與電腦版皆可完美使用。


* 📤 **原生 Web Share 分享**：不經過任何雲端伺服器，直接呼叫作業系統（iOS/Android/macOS/Windows）的原生分享視窗，透過 AirDrop、LINE、Email 等本地管道發送產出的 PDF。
* 🌓 **深色/淺色模式**：支援精美的科技感深色模式與簡約淺色模式，並自動跟隨系統喜好設定。

## 🛠️ 技術棧

* **核心框架**：Next.js 15 (App Router) & React 19 & TypeScript
* **樣式系統**：Tailwind CSS (支援 Dark Mode、磨砂玻璃效果)
* **PDF 解析與渲染**：`pdfjs-dist` (Mozilla PDF.js)
* **PDF 編輯與壓印**：`pdf-lib` (純前端 PDF 合成器)
* **圖示庫**：`lucide-react`
* **特效**：`canvas-confetti` (導出成功時的彩花驚喜)
* **自動部署**：GitHub Actions

## 🚀 本地開發與運行指南

如果您想在本地端執行開發或進行修改，請按照以下步驟操作：

### 1. 安裝 pnpm
本專案嚴格使用 `pnpm` 包管理器。如果您尚未安裝，請執行：
```bash
npm install -g pnpm
```

### 2. 安裝依賴項
在專案根目錄下執行：
```bash
pnpm install
```

### 3. 啟動開發伺服器
```bash
pnpm dev
```
啟動後，在瀏覽器中開啟 [http://localhost:3000](http://localhost:3000) 即可使用。

### 4. 本地打包編譯
進行靜態網頁導出驗證：
```bash
pnpm build
```
Next.js 會將整個專案編譯並導出為靜態網頁，生成於根目錄下的 `./out/` 資料夾中。

## 📦 雲端靜態託管部署

### A. 使用 GitHub Pages 自動部署 (已設定完成)
本專案已配置 GitHub Actions 部署工作流。您只需：
1. 將程式碼推送到 GitHub 的 `main` 或 `master` 分支。
2. 前往 GitHub 倉庫的 **Settings** -> **Pages**。
3. 將 **Build and deployment** 下方的 **Source** 更改為 **`GitHub Actions`**。
4. 推送後，GitHub 系統將自動在雲端完成 `pnpm install` 與編譯，並自動部署上線。

### B. 部署於 Vercel / Cloudflare Pages
1. 將專案推送到 GitHub。
2. 在 Vercel 或 Cloudflare Pages 直接匯入此倉庫。
3. 平台會自動識別 Next.js 專案並在雲端一鍵完成部署。
